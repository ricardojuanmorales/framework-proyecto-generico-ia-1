## 0. Declaración de formalización

El **Caleidoscopio** queda formalizado como lógica integradora emergente del Framework Genérico en versión v0.1. No es un cuarto perfil tradicional ni una suma mecánica de perfiles. Es una lógica de combinación, transformación y emergencia que articula Programador Humanista, Investigador Transdisciplinario y Artista Transdisciplinario cuando un proyecto requiere producir algo que ninguno de los perfiles podría lograr de forma aislada.

El Caleidoscopio opera cuando programación, investigación y creación se afectan mutuamente bajo criterios de evidencia, ética, accesibilidad, documentación, archivo y transferencia.

---

## 1. Definición

El **Caleidoscopio** es la lógica integradora del Framework Genérico que permite combinar perfiles transversales para atender problemas complejos mediante productos emergentes: aplicaciones, recursos educativos abiertos, ambientes virtuales, archivos vivos, publicaciones multimodales, laboratorios, prototipos, políticas, experiencias comunitarias, sistemas documentales, avatares, herramientas, exposiciones o programas institucionales.

No combina perfiles para repartir tareas. Los combina para generar nuevas formas de trabajo.

---

## 2. Principio rector

> La integración transdisciplinaria no ocurre cuando se colocan perfiles uno al lado del otro, sino cuando sus métodos, evidencias, lenguajes, riesgos y formas de validación se transforman mutuamente para producir algo emergente, documentado y transferible.

---

## 3. Pregunta guía

> ¿Qué emerge cuando el Programador Humanista, el Investigador Transdisciplinario y el Artista Transdisciplinario se integran bajo evidencia, ética, accesibilidad, documentación activa, archivo y transferencia?

---

## 4. Problema que atiende

Muchos proyectos complejos fallan porque intentan resolver problemas transdisciplinarios con una sola lógica dominante:

1. técnica sin contexto;
2. investigación sin transferencia;
3. creación sin documentación;
4. innovación sin ética;
5. comunidad sin reciprocidad;
6. educación sin accesibilidad;
7. IA sin gobernanza;
8. archivo sin reactivación.

El Caleidoscopio atiende este vacío creando un flujo donde los perfiles no compiten, sino que se coordinan con criterios comunes de propósito, evidencia, riesgo, derechos, accesibilidad, publicación, archivo y continuidad.

---

## 5. Cuándo activar Caleidoscopio

Activar Caleidoscopio cuando:

- un solo perfil no basta;
- el proyecto requiere investigación y creación;
- el proyecto requiere creación y programación;
- el proyecto requiere investigación y programación;
- hay comunidad, territorio, memoria o institución;
- hay IA significativa;
- hay publicación multimodal;
- hay archivo vivo;
- hay necesidad pedagógica compleja;
- hay transferencia institucional;
- hay riesgos técnicos, epistemológicos, culturales, éticos o socioambientales.

No activar Caleidoscopio cuando la tarea pueda resolverse de forma proporcional con N1 o N2 usando un solo perfil.

---

## 6. Fórmula nuclear

```text
Problema complejo
+ Contexto situado
+ Perfil técnico
+ Perfil investigativo
+ Perfil creativo
+ Ética
+ Accesibilidad
+ Evidencia
+ Documentación
+ Archivo
+ Transferencia
= Producto emergente
```

El Caleidoscopio **no** debe reducirse a:

```text
Programador + Investigador + Artista
```

La integración real exige transformación mutua, no yuxtaposición.

---

## 7. Modos Caleidoscopio C1-C7

| Modo | Integración | Uso típico | Producto emergente |
|---|---|---|---|
| C1 - Técnico-investigativo | Programador + Investigador | apps de investigación, dashboards, pipelines, repositorios | herramienta investigativa validada |
| C2 - Investigativo-creativo | Investigador + Artista | memoria, investigación-creación, publicación multimodal | obra o recurso con evidencia y contexto |
| C3 - Creativo-técnico | Artista + Programador | galerías, avatares, interfaces, instalaciones | experiencia interactiva documentada |
| C4 - Integral | Tres perfiles | AVAT, REA multimodal, ecosistemas agénticos | sistema transdisciplinario completo |
| C5 - Pedagógico | perfiles según necesidad | cursos, módulos, OVAT, trayectorias | experiencia de aprendizaje integral |
| C6 - Comunitario | comunidad + perfiles | archivo vivo, mapas, memoria territorial | plataforma o proceso comunitario cuidado |
| C7 - Institucional | perfiles + gobernanza | laboratorios, programas, políticas, manuales | sistema institucional transferible |

## 8. Descripción de cada modo

### C1 - Técnico-investigativo

Integra Programador Humanista e Investigador Transdisciplinario.

Activar cuando se requiere convertir investigación en herramienta técnica:

- dashboard de evidencia;
- pipeline de análisis;
- repositorio de fuentes;
- aplicación de consulta;
- visualización de datos;
- agente de revisión documental;
- sistema de curaduría o bibliometría.

Evidencia mínima:

```text
pregunta investigativa;
fuentes o corpus;
requisitos técnicos;
prototipo;
validación;
README;
CHECKS técnicos y epistemológicos.
```

---

### C2 - Investigativo-creativo

Integra Investigador Transdisciplinario y Artista Transdisciplinario.

Activar cuando el conocimiento requiere forma estética o cultural:

- investigación-creación;
- memoria histórica;
- archivo narrativo;
- publicación multimodal;
- exposición documental;
- REA creativo;
- experiencia de sensibilización.

Evidencia mínima:

```text

producto multimodal;
accesibilidad;
```

---

- REA multimodal;
- ecosistema de agentes;
- plataforma de aprendizaje;
- archivo vivo interactivo;
- laboratorio de innovación;
- sistema de documentación transdisciplinaria.

Evidencia mínima:

```text
CHECKS;
```

## 9. Flujo integral del Caleidoscopio

```text
Identificar problema complejo
→ Mapear contexto
→ Determinar nivel N1-N4
→ Seleccionar modo C1-C7
→ Activar perfiles necesarios
→ Definir producto emergente esperado
→ Diseñar workflow integrado
→ Identificar riesgos cruzados
→ Producir prototipo o documento
→ Validar con Gates y CHECKS
→ Revisar ética, derechos y accesibilidad
→ Documentar decisiones
→ Actualizar Wiki
→ Cerrar
→ Archivar
→ Transferir / Reactivar
```

Cada fase puede operar como microciclo:

```text
Integrar → Producir → Contrastar → Cuidar → Documentar → Transferir
```

## 10. Matriz de integración por dimensión

| Dimensión | Programador Humanista | Investigador Transdisciplinario | Artista Transdisciplinario | Emergencia Caleidoscopio |
|---|---|---|---|---|
| Problema | Requisitos y sistema | Pregunta y evidencia | Tensión estética y memoria | Problema transdisciplinario situado |
| Contexto | Usuarios y restricciones | Actores, fuentes y territorio | Públicos, símbolos y materialidades | Mapa complejo de sentido y operación |
| Método | Arquitectura y workflow | Metodología y fuentes | Proceso creativo y curatorial | Workflow integrado |
| Producto | App, repositorio, agente | Informe, marco, evidencia | Obra, experiencia, publicación | Producto emergente multimodal |
| Validación | Tests, seguridad, rendimiento | Evidencia, pares, metodología | Recepción, derechos, accesibilidad | Validación multidimensional |
| Documentación | README, código, release | Fuentes, método, síntesis | Bitácora, ficha, archivo | Documentación activa integrada |
| Transferencia | Despliegue y mantenimiento | REA, publicación, capacitación | Mediación, exposición, archivo | Transferencia técnica, cultural y educativa |

## 15. Riesgos principales y salvaguardas

| Riesgo | Descripción | Salvaguarda |
|---|---|---|
| Suma superficial | Poner perfiles juntos sin integración real. | Definir producto emergente y workflow integrado. |
| Dominio técnico | La programación borra contexto, cultura o evidencia. | Gate de contexto y ética. |
| Dominio académico | Investigación queda sin producto transferible. | Gate de transferencia y publicación. |
| Dominio estético | Creación sin documentación o evidencia. | Gate de documentación y derechos. |
| Riesgo cultural | Representación, memoria o símbolos tratados sin cuidado. | Revisión cultural y consentimiento. |
| Riesgo de IA | IA produce sin validación humana. | Bitácora IA y revisión humana. |
| Archivo débil | Producto no puede reactivarse. | Manifest, paquete y primer. |
| Gobernanza difusa | Nadie sabe quién decide. | RACI o registro de responsables. |
