# Guía de Niveles N1-N4

## Propósito

Ayudar a decidir cuánto Framework activar según escala, riesgo y necesidad de trazabilidad.

## N1 — Mínimo controlado

Para tareas pequeñas, borradores simples y bajo riesgo.

**Exige:** propósito, evidencia mínima, cierre breve.

## N2 — Operativo

Para proyectos con varias fases, documentación y revisión básica.

**Exige:** README, matriz simple, checklist, decisiones básicas, continuidad.

## N3 — Avanzado

Para perfiles transversales, Caleidoscopio, IA sustantiva, evaluación, wiki y archivo.

**Exige:** índice, manifest, Gates, CHECKS, WIKI_SYNC, registro, changelog, archivo.

## N4 — Institucional

Para programas, laboratorios, adopción formal, auditoría, capacitación o política.

**Exige:** gobernanza formal, RACI, auditoría, capacitación, mantenimiento, release institucional.

## Cuándo subir de nivel

Subir si hay:

- más actores;
- datos o comunidad sensible;
- IA sostenida;
- publicación pública;
- integración Caleidoscopio;
- mantenimiento esperado;
- transferencia institucional.

## Cuándo bajar de nivel

Bajar si:

- el producto es exploratorio;
- no hay publicación;
- no hay integración real;
- no hay riesgo significativo;
- no hay mantenimiento.
