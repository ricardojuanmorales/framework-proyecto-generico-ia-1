# Guía Gates y CHECKS

## Diferencia

**Gate:** compuerta de validación. Pregunta si se puede avanzar.  
**CHECK:** lista verificable. Dice qué debe revisarse.

## Gates base

```text
G1 Propósito
G2 Ubicación macro-meso-micro
G3 Perfil o Caleidoscopio
G4 Metodología / flujo
G5 Evidencia
G6 IA responsable
G7 Ética, derechos y cuidado
G8 Accesibilidad
G9 Calidad
G10 Documentación activa
G11 Wiki Sync
G12 Archivo
G13 Continuidad
G14 Estabilización
```

## Uso rápido

Antes de avanzar:

```text
¿Qué Gate aplica?
¿Qué evidencia existe?
Quién validó?
Qué documento se actualizó?
Qué riesgo queda?
Puede avanzar?
```

## Estados posibles

```text
Cumple
Cumple con pendiente
No cumple
No aplica
Requiere validación humana
```

## CHECKS mínimos antes de cerrar

- Propósito claro.
- Producto con versión y estado.
- Evidencia verificable.
- IA documentada si aplica.
- Riesgos revisados.
- Accesibilidad revisada.
- Changelog o decisiones actualizados si aplica.
- Wiki_sync evaluado.
- Próximo paso declarado.
