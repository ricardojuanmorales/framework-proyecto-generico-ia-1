# Framework Genérico V4.1.1 — Repositorio Modelo

**Estado:** estructura operativa inicial para GitHub  
**Base aprobada:** Framework Genérico V4.1.0-estable  
**Propósito:** convertir la versión estable en un repositorio vivo, navegable y mantenible.  
**Nivel recomendado:** N3 — Avanzado documental y operativo.  

## Qué contiene

Este repositorio organiza los documentos ya generados del Framework Genérico V4.1.0 y prepara el ciclo menor **V4.1.1**.

Incluye:

- estructura de cartapacios del Framework;
- documentos existentes colocados por función;
- `nada.md` en cartapacios vacíos para que GitHub los conserve;
- README raíz;
- manifest del repositorio;
- documento de apertura V4.1.1.

## Regla de uso

```text
V4.1.0-estable = versión aprobada.
V4.1.1 = ciclo operativo de repositorio, organización, wiki secundaria, piloto y refinamiento.
```

## Cómo comenzar

1. Abrir `00_CONTROL_MAESTRO/README_REPOSITORIO_MODELO_V4_1_1.md`.
2. Revisar `00_CONTROL_MAESTRO/manifest_repositorio_v4_1_1.json`.
3. Consultar documentos oficiales en `00_CONTROL_MAESTRO/`.
4. Trabajar perfiles en `02_ARQUITECTURA_CONCEPTUAL/07_Perfiles_Transversales/`.
5. Registrar cambios en `18_DOCUMENTACION_ACTIVA/`.
6. Preservar releases en `99_ARCHIVO_HISTORICO/`.

## Nota sobre `nada.md`

GitHub no conserva carpetas vacías. Cada cartapacio vacío contiene `nada.md` como marcador. Puede reemplazarse cuando se añada contenido real.
