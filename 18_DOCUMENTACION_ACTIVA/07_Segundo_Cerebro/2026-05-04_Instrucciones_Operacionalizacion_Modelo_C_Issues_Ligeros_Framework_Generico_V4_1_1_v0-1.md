## 7. Paso 6 — Definir flujo de sesión

Usar este flujo en cada sesión de trabajo.

### 7.1 Antes

```text
1. Abrir Dashboard del segundo cerebro.
2. Revisar Issues abiertas.
3. Abrir Primer más reciente.
4. Revisar README y Manifest si cambió el repositorio.
5. Definir objetivo y nivel.
```

### 7.2 Durante

```text
1. Producir documento o revisión.
2. Aplicar Gates.
3. Aplicar CHECKS.
4. Identificar decisiones.
5. Identificar WIKI_SYNC.
6. Identificar archivo o release si aplica.
```

### 7.3 Después

```text
1. Guardar documento en cartapacio correcto.
2. Actualizar Changelog.
3. Actualizar Registro de Decisiones.
4. Actualizar Primer.
5. Actualizar Manifest si cambió estructura.
6. Actualizar Dashboard.
7. Cerrar o actualizar Issue.
8. Hacer commit.
```

---

## 8. Paso 7 — Definir flujo de gobernanza

Actualizar gobernanza cuando ocurra cualquiera de estos eventos:

```text
nuevo documento oficial;
nueva decisión;
cambio de versión;
cambio de estructura;
nuevo release;
nueva wiki;
nuevo perfil;
nuevo Gate o CHECK;
nuevo plan de archivo;
cierre de ciclo.
```

### Documentos que se actualizan

## 9. Paso 8 — Definir flujo de segundo cerebro

Cada vez que se cierre una sesión:

- [ ] Actualizar estado actual.
- [ ] Añadir enlace al documento generado.
- [ ] Actualizar issue correspondiente.
- [ ] Registrar próxima acción.
- [ ] Registrar riesgo nuevo.
- [ ] Registrar decisión nueva.
- [ ] Registrar si requiere wiki.
- [ ] Registrar si requiere archivo.

El segundo cerebro debe guardar interpretación, no duplicar todo.

---

## 10. Paso 9 — Definir flujo de release

Cuando se cierre un ciclo:

```text
1. Aplicar CHECKS de archivo.
2. Crear Release Notes.
3. Actualizar manifest.
4. Actualizar índice.
5. Crear ZIP de release.
6. Registrar decisión.
7. Actualizar Dashboard.
8. Cerrar Issues del ciclo.
```

---

## 11. Paso 10 — Semáforo de monitoreo

Usar este semáforo en el Dashboard:

| Color | Estado | Acción |
|---|---|---|
| Verde | Todo sincronizado | Continuar |
| Amarillo | Hay pendientes de wiki, issues o manifest | Actualizar antes de release |
| Rojo | Hay decisión sin registrar o documento sin versión | Detener y corregir |
| Azul | Listo para release | Preparar cierre |
| Morado | Listo para N4 | Preparar auditoría, capacitación y mantenimiento |
