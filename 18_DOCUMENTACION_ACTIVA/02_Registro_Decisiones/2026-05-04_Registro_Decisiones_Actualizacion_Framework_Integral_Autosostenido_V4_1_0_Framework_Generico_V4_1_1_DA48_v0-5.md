---

## 2. Justificación

El proyecto necesita una versión compacta, integral y autosostenida para que colegas puedan:

```text
comprender el framework;
diagnosticar necesidades del usuario;
determinar nivel N1-N4;
activar perfiles transversales;
usar Caleidoscopio;
aplicar Gates y CHECKS;
crear documentación mínima;
cerrar y archivar sesiones;
probar el framework en proyectos propios.
```

Sin esta pieza, el framework dependería demasiado del historial conversacional, de V4.0.0 o de múltiples documentos distribuidos.
