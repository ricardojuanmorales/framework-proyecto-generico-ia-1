## 1. Propósito

## 2. Decisiones registradas

### D-A24 — Paquete Consolidado de Perfiles Transversales y Caleidoscopio

**Decisión:** Crear el Paquete Consolidado de Perfiles Transversales y Caleidoscopio v0.1 como artefacto derivado de navegación, transferencia y activación.

**Tipo de cambio:** Ajuste menor documental + extensión modular operativa.  
**Justificación:** Los cuatro componentes ya estaban formalizados y requerían un artefacto de navegación común.  
**Impacto:** Mejora continuidad, lectura transversal, transferencia y preparación de V4.1.0.  
**Riesgo:** Duplicar o reescribir perfiles base.  
**Salvaguarda:** Declarar el paquete como documento derivado, no sustituto.  
**Documentos afectados:** Compendio, Primer, README de Perfiles Transversales, Changelog.  
**Versión afectada:** V4.0.0 / capa transversal v0.1.  
**Estado:** Aceptada y ejecutada.

---

### D-A25 — Matriz Comparativa Transversal

**Decisión:** Crear la Matriz Comparativa Transversal v0.1 como documento derivado del paquete consolidado.

**Tipo de cambio:** Extensión modular.  
**Justificación:** Se necesitaba comparar propósito, flujo, capacidades, evidencias, riesgos, sistemas documentales, productos, wiki y niveles N1–N4.  
**Impacto:** Facilita activación selectiva de perfiles y evita confusión entre componentes.  
**Riesgo:** Simplificar diferencias profundas entre perfiles.  
**Salvaguarda:** Mantener trazabilidad hacia documentos base.  
**Documentos afectados:** Paquete consolidado, Primer, Wiki humana.  
**Versión afectada:** Perfiles Transversales v0.1.  
**Estado:** Aceptada y ejecutada.

---

### D-A26 — Matriz Caleidoscopio C1–C7 con Ejemplos Operativos

**Decisión:** Crear la Matriz Caleidoscopio C1–C7 con ejemplos operativos como extensión modular del Marco Caleidoscopio v0.1.

**Tipo de cambio:** Extensión modular operativa.  
**Justificación:** El Caleidoscopio necesitaba ejemplos concretos para demostrar emergencia transdisciplinaria y no mera suma de perfiles.  
**Impacto:** Convierte C1–C7 en rutas aplicables a proyectos reales.  
**Riesgo:** Producir ejemplos genéricos o decorativos.  
**Salvaguarda:** Exigir problema, contexto, perfiles, fundamentos, metodologías, agentes, skills, evidencia, Gates, riesgos, accesibilidad, derechos, licencias y continuidad.  
**Documentos afectados:** Marco Caleidoscopio, Paquete Consolidado, Wiki Caleidoscopio.  
**Versión afectada:** Caleidoscopio v0.1.  
**Estado:** Aceptada y ejecutada.

---

### D-A27 — Plantillas Base Agents, Skills, Workflows, Checks y Wiki Sync

**Decisión:** Crear las Plantillas Base Agents, Skills, Workflows, Checks y Wiki Sync v0.1 como extensión modular operativa.

**Tipo de cambio:** Extensión modular operativa.  
**Justificación:** Los perfiles requieren moldes reutilizables para activar agentes, skills, workflows, verificaciones y sincronización documental sin automatización prematura.  
**Impacto:** Prepara operación humano-IA documentada y transferible.  
**Riesgo:** Convertir plantillas en automatización antes de validación.  
**Salvaguarda:** Mantenerlas como plantillas base hasta que existan pilotos.  
**Documentos afectados:** Sistema Operativo Agents/Skills, Wiki humana, CHECKS, WIKI_SYNC.  
**Versión afectada:** V4.0.0 / preparación V4.1.0.  
**Estado:** Aceptada y ejecutada.

---

### D-A28 — Descongelamiento Controlado V4.0.0 y Apertura Borrador V4.1.0

**Decisión:** Descongelar de forma controlada el Framework Genérico V4.0.0 y abrir el borrador Framework Genérico V4.1.0 para integrar formalmente la capa de Perfiles Transversales, Caleidoscopio, matrices, ejemplos operativos, plantillas base y gobernanza documental asociada.

**Tipo de cambio:** Cambio de gobernanza + cambio metodológico.  
**Justificación:** La V4.0.0 alcanzó madurez funcional y la capa transversal ya cuenta con productos suficientes para integrarse en una versión evolutiva.  
**Impacto:** Alto. Permite pasar de arquitectura estabilizada a borrador operativo V4.1.0.  
**Riesgo:** Confundir evolución con reescritura total.  
**Salvaguarda:** Cerrar V4.0.0 formalmente, preservar documentos base y abrir V4.1.0 como borrador, no release final.  
**Documentos afectados:** Estado Actual, Changelog, Registro de Decisiones, Primer, README, Wiki, Archivo Histórico.  
**Versión afectada:** V4.0.0 → V4.1.0-borrador.  
**Estado:** Aceptada por validación humana y lista para ejecución.

---

## 3. Reglas de continuidad

1. No reescribir perfiles cerrados sin decisión explícita.
2. No crear macrocarpetas nuevas sin mapa y análisis de impacto.
3. Primero mapa, luego documento, luego carpeta, luego automatización.
4. Toda automatización futura debe pasar por checks, Gates y WIKI_SYNC.
5. Caleidoscopio debe mantenerse como lógica integradora emergente.
6. Todo cambio debe registrar justificación, impacto, riesgo, documentos afectados, versión y continuidad.
