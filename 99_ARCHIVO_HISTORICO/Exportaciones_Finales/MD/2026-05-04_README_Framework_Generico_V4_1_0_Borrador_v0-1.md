## 0. Declaración de apertura

El **Framework Genérico V4.1.0-borrador** se abre como evolución controlada del **Framework Genérico V4.0.0**.

No es una reescritura total.  
No sustituye de inmediato a V4.0.0.  
No elimina documentos base.  
No crea macroarquitectura nueva sin justificación.

Su función es incorporar formalmente la capa de **Perfiles Transversales y Caleidoscopio** desarrollada durante el ciclo de consolidación V4.0.0, junto con sus matrices, ejemplos operativos, plantillas base, gobernanza documental, cierre histórico y continuidad.

La V4.1.0-borrador debe entenderse como una **rama evolutiva trazable**: una puerta de actualización, no una demolición.

## 3. Qué cambia en V4.1.0-borrador

La V4.1.0-borrador incorpora formalmente una capa transversal operativa que permite activar perfiles, combinarlos y documentar productos emergentes.

### 3.1 Cambios principales

| Cambio | Tipo | Estado |
|---|---|---|
| Integración formal de Perfiles Transversales | Extensión modular | Abierta |
| Integración formal de Caleidoscopio | Extensión metodológica | Abierta |
| Incorporación de matrices comparativas | Ajuste documental | Abierta |
| Incorporación de ejemplos C1-C7 | Extensión operativa | Abierta |
| Incorporación de plantillas agents/skills/workflows/checks/wiki_sync | Extensión operativa | Abierta |
| Actualización de gobernanza documental | Cambio de gobernanza menor | En curso |
| Preparación de archivo histórico V4.0.0 | Cierre documental | En curso |
| Preparación de ruta V4.1.0 | Apertura evolutiva | En curso |

### 3.2 Qué no cambia todavía

```text
No se reescribe la macroarquitectura completa.
No se reemplaza el Framework V4.0.0 como versión estable.
No se cierran Investigador v0.2 ni Artista v0.2 todavía.
No se convierte Caleidoscopio en un cuarto perfil tradicional.
No se automatizan agentes antes de validar plantillas, checks y evidencia.
No se crean nuevas macrocarpetas sin mapa, decisión y justificación.
```

## 4. Núcleo conceptual de V4.1.0-borrador

La V4.1.0-borrador se organiza alrededor de esta tesis operativa:

> El Framework Genérico evoluciona cuando logra convertir su arquitectura madre en un sistema vivo de perfiles, combinaciones, evidencias, agentes, documentación, evaluación y transferencia sin perder juicio humano, ética, accesibilidad ni trazabilidad.

La nueva versión trabaja con cuatro componentes transversales:

```text
Programador Humanista

```

## 5. Perfiles Transversales

### 5.1 Programador Humanista

**Versión:** v0.2  
**Estado:** cerrado funcional  
**Función:** problematizar, arquitecturar, diseñar, programar, probar, validar, documentar, cerrar, archivar y transferir sistemas sociotécnicos.

Fórmula nuclear:

```text
Problema
+ Contexto
+ Requisitos
+ Arquitectura
+ Diseño
+ Código
+ Prueba
+ Validación
+ Documentación
+ Cierre
+ Archivo
+ Transferencia
```

Regla: no se reescribe sin decisión explícita.

---

### 5.2 Investigador Transdisciplinario

**Versión:** v0.1  
**Estado:** formalizado preliminar  
**Función:** producir conocimiento situado, validable, documentado y transferible mediante saberes, métodos, fuentes, evidencia, IA responsable, publicación multimodal y REA.

Fórmula nuclear:

```text
Problema
+ Contexto
+ Saberes

+ Metodología
+ Fuentes
+ Evidencia
+ Análisis
+ Síntesis
+ Validación
+ Comunicación
+ Publicación
+ Documentación
+ Archivo
+ Transferencia
```

Regla: puede evolucionar hacia v0.2 con referencias, validación y decisión explícita.

---

### 5.3 Artista Transdisciplinario

**Versión:** v0.1  
**Estado:** formalizado preliminar  
**Función:** crear, documentar, curar, publicar, archivar y transferir obras, narrativas, avatares, colecciones y experiencias multimodales con cuidado cultural, derechos, accesibilidad y memoria.

Fórmula nuclear:

```text
Contexto
+ Memoria
+ Saberes
+ Símbolos
+ Materialidades
+ Concepto
+ Prototipo
+ Creación
+ Curaduría
+ Documentación
+ Publicación / Exposición
+ Archivo
+ Transferencia
```

Regla: puede evolucionar hacia v0.2 con referencias, validación y decisión explícita.

---

### 5.4 Caleidoscopio

**Versión:** v0.1  
**Estado:** lógica integradora preliminar  
**Función:** combinar perfiles y producir emergencia transdisciplinaria con evidencia, ética, accesibilidad, evaluación, documentación, archivo y transferencia.

Fórmula nuclear:

```text
Problema complejo
+ Contexto situado
+ Saberes diversos
+ Investigación rigurosa
+ Creación sensible
+ Programación responsable
+ IA gobernada
+ Prototipado
+ Evaluación
+ Publicación multimodal
+ Documentación activa
+ Archivo
+ Transferencia
= Emergencia transdisciplinaria
```

Regla: no debe tratarse como perfil aislado ni como simple suma de perfiles.

## 6. Modos Caleidoscopio C1-C7

| Modo | Combinación | Uso principal | Producto típico |
|---|---|---|---|
| C1 — Técnico-investigativo | Programador + Investigador | Apps, dashboards, bibliometría, repositorios, sistemas agénticos | Herramienta, pipeline, app, visualización |
| C2 — Investigativo-creativo | Investigador + Artista | Investigación-creación, memoria, narrativa, documentación cultural | Obra documentada, publicación multimodal |
| C3 — Creativo-técnico | Artista + Programador | Arte digital, galería interactiva, avatar, experiencia inmersiva | Prototipo, interfaz, instalación digital |
| C4 — Integral | Programador + Investigador + Artista | Investigación, creación, tecnología y transferencia | AVAT, REA multimodal, ecosistema agéntico |
| C5 — Pedagógico | Investigador + Artista + Programador si aplica | Cursos, módulos, OVAT, rutas formativas | Curso virtual, módulo REA |
| C6 — Comunitario | Artista + Investigador + comunidad + Programador si aplica | Memoria, archivo, territorio, participación | Archivo vivo, galería comunitaria |
| C7 — Institucional | Tres perfiles + gobernanza | Transferencia, política, capacitación, escalamiento | Programa, laboratorio, manual, wiki |

| Nivel | Uso de V4.1.0-borrador |

| N1 — Mínimo controlado | Activar un perfil, producir evidencia mínima, documentar y cerrar. |

| N4 — Institucional | Formalizar gobernanza, capacitación, repositorios, dashboards, archivo histórico, transferencia y adopción. |

Regla: ningún nivel elimina juicio humano, documentación activa ni cierre.

## 13. Riesgos principales

| Riesgo | Descripción | Salvaguarda |
|---|---|---|
| Reescritura de V4.0.0 | Confundir evolución con sustitución. | Mantener V4.0.0 archivada y citada. |
| Proliferación documental | Crear documentos sin función clara. | Aplicar regla mapa → documento → carpeta → automatización. |
| Caleidoscopio reducido | Usarlo como suma de perfiles. | Exigir emergencia, evidencia y validación integrada. |
| Automatización prematura | Crear agentes sin checks ni gobernanza. | Validar plantillas antes de automatizar. |
| Wiki tardía | Dejar guías humanas para después. | Activar WIKI_SYNC en cada cambio sustantivo. |
| Falta de cierre | Avanzar sin changelog, decisiones ni primer. | Cierre documental obligatorio. |
| Pérdida de juicio humano | Delegar validación a IA. | Mantener validación humana explícita. |
