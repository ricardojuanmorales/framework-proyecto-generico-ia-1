## 0. Declaración de consolidación

Este paquete consolida la capa de **Perfiles Transversales y Caleidoscopio** del Framework Genérico V4.0.0 como sistema de navegación, activación y transferencia.

Integra cuatro componentes ya formalizados:

```text
01_Programador_Humanista_v0-2
02_Investigador_Transdisciplinario_v0-1
03_Artista_Transdisciplinario_v0-1
04_Caleidoscopio_v0-1
```

El paquete no crea una nueva macroarquitectura. No convierte los perfiles en carpetas principales nuevas. No reabre el Programador Humanista v0.2. Su función es ordenar, comparar, conectar y preparar la activación práctica de los perfiles en proyectos reales.

Principio rector:

> Consolidar no es mezclar. Consolidar es hacer visible la relación entre componentes ya formalizados para que puedan operar con trazabilidad, ética, accesibilidad, evaluación, documentación, cierre, archivo y continuidad.

## 4. Síntesis de los perfiles

### 4.1 Programador Humanista

**Función central:** construir, validar, documentar, mantener y transferir sistemas sociotécnicos con juicio humano, ética, accesibilidad, seguridad y trazabilidad.

**Fórmula nuclear:**

```text
Problema
+ Contexto
+ Requisitos
+ Arquitectura
+ Diseño
+ Código
+ Prueba
+ Validación
+ Documentación
+ Cierre
+ Archivo
+ Transferencia
```

**No debe reducirse a:**

```text
Código + IA + Automatización
```

**Producto típico:** aplicación, repositorio, pipeline, sistema documental, dashboard, prototipo técnico, agente, flujo automatizado, wiki técnica o paquete de despliegue.

**Evidencia mínima:** requisitos, arquitectura, código o prototipo, pruebas, documentación, bitácora, changelog, decisión registrada y cierre.

---

### 4.2 Investigador Transdisciplinario

**Función central:** formular problemas complejos, integrar saberes, diseñar rutas de indagación, gestionar fuentes, analizar evidencia, validar conocimiento y transferir resultados.

**Fórmula nuclear:**

```text
Problema
+ Contexto
+ Saberes
+ Marcos conceptuales
+ Metodología
+ Fuentes
+ Evidencia
+ Análisis
+ Síntesis
+ Validación
+ Comunicación
+ Publicación
+ Documentación
+ Archivo
+ Transferencia
```

**No debe reducirse a:**

```text
Búsqueda + IA + Resumen
```

**Producto típico:** revisión de literatura, bibliografía anotada, marco conceptual, corpus, informe, publicación multimodal, REA, repositorio investigativo, ruta AVAT/OVAT o guía de transferencia.

**Evidencia mínima:** problema, pregunta guía, mapa de contexto, fuente registrada, criterio metodológico, síntesis, validación humana, documentación y cierre.

---

### 4.3 Artista Transdisciplinario

**Función central:** crear, documentar, curar y transferir obras, experiencias, narrativas, colecciones, avatares, ambientes y publicaciones multimodales con sensibilidad cultural, ética, accesibilidad y archivo.

**Fórmula nuclear:**

```text
Contexto
+ Memoria
+ Saberes
+ Símbolos
+ Materialidades
+ Concepto
+ Prototipo
+ Creación
+ Curaduría
+ Documentación
+ Publicación / Exposición
+ Archivo
+ Transferencia
```

**No debe reducirse a:**

```text
Imagen + Estilo + Producción rápida
```

**Producto típico:** obra, serie visual, instalación, avatar, galería, exposición, narrativa multimodal, archivo cultural, colección documentada, experiencia estética, recurso educativo abierto o mediación comunitaria.

**Evidencia mínima:** contexto, statement, bitácora creativa, ficha de obra, derechos/licencias, accesibilidad, documentación curatorial, archivo y cierre.

---

### 4.4 Caleidoscopio

**Función central:** integrar los perfiles para producir emergencia transdisciplinaria: aquello que no aparece cuando cada perfil trabaja por separado.

**Fórmula nuclear:**

```text
Problema complejo
+ Contexto situado
+ Saberes diversos
+ Investigación rigurosa
+ Creación sensible
+ Programación responsable
+ IA gobernada
+ Prototipado
+ Evaluación
+ Publicación multimodal
+ Documentación activa
+ Archivo
+ Transferencia
= Emergencia transdisciplinaria
```

**No debe reducirse a:**

```text
Tres perfiles juntos
```

**Producto típico:** AVAT, REA multimodal, ecosistema agéntico, archivo vivo, aplicación cultural, galería interactiva, laboratorio, programa institucional, repositorio, narrativa computacional o experiencia comunitaria documentada.

**Evidencia mínima:** problema complejo, modo C1-C7, matriz de combinación, fundamentos integrados, metodología, prototipo, validación multidimensional, documentación, publicación, archivo y continuidad.

```text
                         Caleidoscopio
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
Programador Humanista   Investigador          Artista
                         Transdisciplinario    Transdisciplinario
        │                     │                     │
Construcción técnica     Evidencia situada      Creación sensible
Arquitectura             Metodología            Memoria / símbolos
Código / pruebas         Fuentes / análisis     Curaduría / obra
Seguridad                Validación             Derechos / licencias
Despliegue               Publicación            Exposición / archivo
Documentación            Transferencia          Mediación cultural
```

Lectura operativa:

```text
El Programador Humanista hace construible.
El Investigador Transdisciplinario hace validable.
El Artista Transdisciplinario hace sensible y comunicable.
El Caleidoscopio hace emergente, integrable y transferible.
```

## 6. Ruta de uso por nivel N1-N4

| Nivel | Uso del paquete | Documentación mínima | Producto esperado |
|---|---|---|---|
| N1 — Mínimo controlado | Activar uno o dos perfiles con propósito claro. | Brief, evidencia mínima, cierre. | Prototipo simple, síntesis, obra breve o recurso inicial. |
| N2 — Operativo | Comparar perfiles y seleccionar combinación. | Matriz breve, workflow, checklist, wiki mínima. | Producto documentado y validado inicialmente. |
| N3 — Avanzado | Activar Caleidoscopio con modos C1-C7, Gates y sistemas documentales. | Agents/skills/workflows/checks/wiki_sync, changelog, decisiones. | Prototipo integrado, publicación multimodal, REA, AVAT o repositorio. |
| N4 — Institucional | Escalar a gobernanza, capacitación, auditoría y transferencia formal. | Manifest, indicadores, archivo histórico, wiki institucional, plan de adopción. | Programa, laboratorio, política, archivo vivo o ecosistema institucional. |

Regla:

> Ningún nivel elimina juicio humano, evidencia, ética, accesibilidad, documentación, cierre ni archivo. Solo cambia la profundidad del proceso.

## 7. Modos Caleidoscopio C1-C7

| Modo | Combinación | Cuándo se activa | Producto típico |
|---|---|---|---|
| C1 — Técnico-investigativo | Programador + Investigador | Cuando el problema requiere evidencia, datos, fuentes, repositorio, pipeline o visualización. | App, dashboard, bibliometría, repositorio, sistema agéntico. |
| C2 — Investigativo-creativo | Investigador + Artista | Cuando el problema requiere memoria, narrativa, cultura, interpretación, obra o exposición documentada. | Investigación-creación, publicación multimodal, catálogo, exposición. |
| C3 — Creativo-técnico | Artista + Programador | Cuando el problema requiere experiencia interactiva, avatar, interfaz, instalación o prototipo digital. | Galería interactiva, avatar, app creativa, instalación digital. |
| C4 — Integral | Programador + Investigador + Artista | Cuando el problema necesita evidencia, creación, tecnología, evaluación y transferencia. | AVAT, REA multimodal, ecosistema agéntico, archivo vivo. |
| C5 — Pedagógico | Investigador + Artista + Programador si aplica | Cuando el producto principal es formativo. | Curso, módulo, OVAT, ruta de aprendizaje, microcredencial. |
| C6 — Comunitario | Artista + Investigador + comunidad + Programador si aplica | Cuando el centro es memoria, territorio, participación o archivo vivo. | Archivo comunitario, mapa interactivo, galería territorial. |
| C7 — Institucional | Tres perfiles + gobernanza | Cuando se busca escalamiento, política, capacitación o adopción institucional. | Laboratorio, manual, wiki institucional, programa, política. |
